# 미션 컴퓨터 사고 분석 보고서 (Log Analysis Report)

## 1. 개요
본 보고서는 미션 컴퓨터 로그 파일(`mission_computer_main.log`)을 분석하여 화성 기지 폭발 사고의 선행 사건과 직접적인 원인을 파악하기 위해 작성되었습니다.

## 2. 분석 환경
- **분석 도구**: Python 3.x (사용자 정의 로그 분석 스크립트)
- **대상 파일**: mission_computer_main.log
- **인코딩**: UTF-8

## 3. 타임라인 분석
| 시간 (Timestamp) | 구분 | 주요 사건 (Event) |
| :--- | :--- | :--- |
| 10:00 ~ 11:30 | 정상 | 로켓 발사, 궤도 진입, 위성 배포 및 착륙 성공 |
| 11:30:00 | 완료 | 미션 성공 완료 선언 및 회수팀 파견 |
| **11:35:00** | **주의** | **산소 탱크 불안정 현상 감지 (Oxygen tank unstable)** |
| **11:40:00** | **사고** | **산소 탱크 폭발 (Oxygen tank explosion)** |
| 12:00:00 | 종료 | 기지 및 제어 시스템 전원 차단 |

## 4. 사고 원인 결론
로그 분석 결과, 화성 기지 폭발의 직접적인 원인은 **산소 탱크의 불안정성으로 인한 폭발**로 판명되었습니다. 

미션이 성공적으로 완료된 시점(11:30)으로부터 5분 후인 **11:35**에 산소 탱크에서 이상 징후가 처음 발견되었으며, 그로부터 다시 5분 후인 **11:40**에 폭발이 발생했습니다. 

## 5. 향후 조치 제안
- 산소 탱크 계통의 설계 결함 여부 조사
- 비상 전력 공급 장치 및 산소 탱크 모니터링 시스템의 독립성 확보
- 재건 시 폭발 방지 및 피해 최소화를 위한 격벽 구조 강화
